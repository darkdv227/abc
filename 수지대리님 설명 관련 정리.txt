	* 배송 업무 최우선 처리
		주문 -> 배송 -> 취소, 반품, 클레임 등 후속 비즈니스가 물려있기 때문에 배송 업무 처리가 최우선적으로 해결되어야 한다.
		
	* 화면 1순위 처리 
		1. 	제휴사 상품 목록 
				제휴사 상품 목록 화면은 다른 파트에서 조회가 필요한 화면이기 때문에 화면쪽에서 최우선으로 처리할 필요가 있다.
				
				1. 목록 조회/ 상품 저장 
					- 상품 신규등록배치 대상 저장(유효성체크 포함)
						하얀색 - 신규 등록으로 12/24시 등록 배치 수행까지 신규등록 대기 상태
						빨간색 - 등록 실패로 상품정보 및 제휴사 정책 확인 필요 상태
						파란색 - 신규 등록 성공
						노란색 - 제휴사 연동해지 상태
					  
				2. 신규등록가이드 팝업
				3. 상품삭제 팝업
					- 제휴사 클릭 저장(adm)
					- OB 판매중지 API 호출(OB)
					- 삭제 이력 테이블 저장(ADM)
					- 선택 제휴사 상품관련 테이블 전체 초기화(ADM)
				4. 연동제외 해제
					- 연동제외 체크박스 선택되어 있으면 연동제외 상태
					- 연동제외 체크박스를 체크 해제한 뒤 저장하고 확인
				
		2. 	공통 코드를 조회해 오는 경우에는 화면에서 따로 수정없이 적용되는 케이스들이 다수 존재
				공통 코드로 호출되어 처리되는 부분과 직접적으로 호출하여 처리되는 부분을 구분하고
				직접적으로 처리되는 부분은 작업이 요구된다.
		
		3. 	제휴사 상품가격 목록
				GD, UT 의 두 가지로 구분
				GD(Goods)를 수정하면 전체 옵션에 반영되어 수정된다.
				
				UT(Util)는 그리드 화면에서는 수정할 수 없다.
				수정 팝업에서 수정이 가능하다.
				
				상품의 가격과 관련된 부분은 NS 담당자 쪽과 협의가 필요한 부분이기 때문에, 변경을 위해서는 이야기를 주고 받을 필요가 있다.
				
		4. 	제휴사 상품 등록 취소 화면
				기존 로직과 큰 차이는 없을 것으로 보이므로, 인터파크 소스를 확인하도록 한다.
		
		5. 	제휴사 주문연동 목록
				그리드 쪽의 주문 재연동 버튼
					
				주문 실패 건(재고부족, 상품상태, 단품상태)에 대한 재연동을 수행
				상품 > 상품관리 > 상품조회 > 상세조회 에서 주문 실패원인을 제거해야한다.
					1) 재고부족 : 재고 입력
					2) 상품상태 : 상품상태 ON(판매중) 으로 변경
					3) 단품상태 : 단품상태 ON(판매중) 으로 변경
				
				재연동할 주문 체크 후 주문재연동 버튼 클릭
				
				배치가 주문 재연동을 진행한다. (매시 05분, 25분, 45분 진행)
				
				배치가 돌아간 이후 목록 재확인을 수행한다.
					1) 주문이 목록에 보이지 않으면 주문 연동 성공
					2) 주문이 목록에 보이면 실패 원인 확인
					
				주문 연동 실패 후 취소된 주문을 재연동했을 경우, CS 팀에 요청하여 주문 취소를 진행해야 한다.
					
		6. 	제휴사 배송오류내역 목록
				제휴사 목록이 공통코드로 불러와 지는지 확인
				
				
		7. 	MD별 설정
				'방글이' 로 검색하여 조회
				
				그리드 우측 설정을 클릭하여 나오는 MD별 설정 팝업에서 제휴사 select box에 네이버가 추가되도록 설정
				
		8.	제휴사 클레임 수집대상 설정(SQL)
				권한관리 탭의 제휴사 클레임 수집대상 설정(SQL)
				
				제휴사에 네이버 제대로 추가되는지 확인
				API는 업무단위로 끊어서 구분하기 애매한 부분이 있어 취소/반품/교환 선택에 따른 구분은 되지 않고있다.
